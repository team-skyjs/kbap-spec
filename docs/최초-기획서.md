# 외국인 대상 app 최초 기획서

프로토타입 링크 : https://claude.ai/public/artifacts/7199e787-0690-44c5-9816-38a57ba1c41d

서비스명 : (미정)

타겟 : 한국여행/거주 외국인

서비스 : 회원 가입시 개인 알러지/종교(못 먹는 고기 등) 정보 입력 -> 한식 메뉴판을 찍으면 파파고/구글렌즈같은 단순 번역 뿐아니라 아래와 같은 기능을 제공

1. 개인 알러지 위험도 정보, 메뉴 시킬때 꼭 물어봐야할점 등 제공
    1. 김치찌개에 -> 고추가루 100%들어가는데 해당 알러지가 있으니 시키지 말아라. + 새우젓 등이 매장별로 들어갈 수도 있다, 너는 새우 알러지가 있으니 사장님한테 이렇게 물어봐라~ 라는 UI 제공

2. 메뉴 해설/사진/리뷰 모아서 보여주기

** 중요한 점 : ‘식당’별로 음식을 모아서 보여주는게 아니라 ‘음식’별로 모아서 리뷰를 달고 정보를 나누는 앱임. 기존 네이버맵, 구글맵, 파파고, 구글렌즈와 차별점. 따라서 지도나 식당 자체는 크게 중요하지 않음(나중에 넣을 수는 있지만 현재는 고려 안함)

## 앱 플로우

1. 회원가입
    1. (필수) 이메일 인증, 닉네임, 국적, 언어
    2. (필수) 알러지, 비건/채식, 종교 사항 등 못 먹는 음식 등 구체적으로 입력
    3. (선택) 맵기 정도는 어느정도까지 괜찮은지 입력
        1. 이때 맵기는 앱 전반적으로 0~10단계로 표시하되 7은 불닭볶음면, n단계는 하바네로 고추, nn 단계는 태국 고추 등 외국인 친화적인 비유 포함하여 정의
    4. (선택) 관심 있거나 먹어본 한국음식 선택
        1. 스포티파이 등의 최초 회원가입 화면처럼 인기순으로 최초 9개 음식 표시 > 클릭하면 연관성 있는 9개 추가 표시 하는 식의 ui 차용
2. 앱 진입
    1. 내비게이션 바 탭은 5개 - 홈, 음식, 카메라(렌즈), 커뮤니티, 프로필 (이 순서 그대로, 가운데 카메라는 크고 동그랗게 강조)
    2. 헤더(유튜브와 동일) - 좌측 로고, 우측에는 알림, 검색
3. 카메라(렌즈) 탭
    1. 첫 화면
        1. 기본으로 카메라, 좌 하단 갤러리 탭하면 사진 업로드 가능
        2. 상단에 탭 전환으로 메뉴판, 가공식품, 음식 3가지 중 선택 가능(기본은 메뉴판)
        3. 사진 촬영/업로드 시 위아래로 움직이며 스캔하는 ui로 로딩 표시
    2. 촬영/업로드 후
        1. 메뉴판 → 파파고/구글렌즈처럼 메뉴판 위에 설정 언어로 번역하되 우측에 안전(초록 체크), 주의(노랑 느낌표), 위험(빨강 엑스)로 먹어도 되는지 위험 정도를 표시 → 메뉴 하나 클릭 → 음식 디테일 페이지 bottom sheet 형태로 표시
        2. 가공식품 → 바로 음식 디테일 페이지 표시
        3. 음식 → 음식 파악 정확도 90퍼 이상이면 바로 음식 디테일 페이지, 그 이하면 예상 음식 리스트 주고 선택하는 ui 표시
    3. 음식 디테일 페이지 (아래를 화면 위에서 아래부터 순서대로)
        1. 메뉴명 해당 언어로 최상단 표시
        2. 유저 전체 평점 표시/동일 국적 사람들의 평점 표시, 클릭시 리뷰 디테일 화면 진입(리뷰 디테일 화면은 4-b참고)
        3. description 표시 → 단순 번역 수준이 아닌, 실제로 한국인들이 얼마나 먹는지 대략 어떤 느낌의 음식인지 알 수 있는 정도의 수준이되 영어 기준 150자 이내로
        4. 음식 사진
        5. 90프로 이상 들어가는 재료/성분 표 중 b에서 판단된 위험/주의/안전 순서대로 표시
        6. 이때 ‘주의’로 표시된 재료/성분을 클릭하면, ‘${메뉴명}에는 ${재료/성분}이 ${percentage}% 로 들어가있어요. 정확히 확인하려면 아래 버튼을 누른 뒤 사장님께 화면을 보여주세요’ → 버튼 클릭하면 한글로 ‘${메뉴명}(이때 이 메뉴명은 메뉴판의 메뉴명과 동일해야함)에 ${재료} 가 들어가나요?’ 크게 표시, 하단에 중간 크기 글씨로 ‘제가 알러지가 있어서 먹으면 위험합니다’ 이런식으로 description
4. 음식 탭
    1. 3-c 음식 디테일 페이지를 탐색하기 쉬운 형태로 보여줌
    2. 리뷰 디테일 페이지
        1. 최상단에 전체 유저평점/ 내 국적 유저 평점 보여주기
        2. 리뷰별로 유저의 국적(국기), 랭킹(5-c 참고) 우측에 보여야함
        3. 클릭시 내 언어로 번역가능해야함
        4. 내 국적 리뷰만 모아보기 기능 가능해야함
5. 프로필
    1. 본인 정보 CRUD
    2. 내가 업로드한 리뷰 모아보기
    3. 랭킹 정보
        1. 입문자-뼛속까지 한국인 이런 식으로 유쾌하게 랭킹 표시
6. 커뮤니티
    1. 게시판 기본 형식. 글 제목, 내용, 사진, 댓글, 대댓글, 좋아요
    2. 단 내 국적 사람들 글만 보는 기능이 있어야함
    3. 게시판에 글 남긴/댓글 남긴 모든 사람들의 국기가 보여야함

** 카메라 → 디테일 진입 했던 음식들 저장 후, 30분 뒤 push 알림으로 ‘이 음식을 드셨나요? 리뷰를 남겨주세요!’ 보냄 → 앱 진입 시 디테일 화면 진입 했던 음식들 보여주고 먹은 음식있는지 선택 후 리뷰 작성가능

# 계획
- 1차 mvp : 메뉴판의 알러지/종교/비건 등 개인화 정보만, 리뷰, 애플/구글/자체 회원가입 (~7/10일까지 개발 완료)
- 2차 : 가공 식품, 커뮤니티
- 3차: 길거리 음식 사진찍어서 판별


### 이 외 주의할 점
- 다국어로 지원해야하기 때문에 한국어로 하드코딩하지 않도록 초기 설계 잘 할 것
- 지원할 언어는 방문 많은 국가 우선 순위로 영어 우선, 동남아에서도 제일 인구 수 많은 곳 부터
