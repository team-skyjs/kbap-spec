# K-Bap MVP 출시 커맨드센터

> **역할 분리(2026-07-06 확정)**: 개별 **이슈 상태의 정본은 Jira(KB)**. 이 파일은 **출시 커맨드센터** —
> 타임라인·크리티컬패스·리스크·정책 링크·회의 결정·에픽 맵만 관리하고, per-이슈 체크박스로 상태를 복제하지 않는다.
> "지금 뭐가 남았지?"의 **세부 상태는 Jira**, "출시까지 큰 그림/의존/리스크"는 여기.

> **타임라인**: 개발완료 **7/10(금)** → QA 5일(7/11~14) → 앱 제출 **7/15** → **최종 목표: 7월 내 스토어 다운로드 가능.**
> **API SSOT**: BE Swagger = 진리 (`https://meogo.handev.site/swagger-ui`). 이 레포 `contracts/openapi.yaml`은 참고용(정본 아님).
> **완료 게이트**: 구현 세션은 완료 시 Jira `검토 중`으로만 전환 + 진행로그 댓글. `완료` 전환·BE플래그는 리뷰(예진+Claude) 후. (`.claude/skills/jira` §5)

## 🎯 크리티컬 패스 (출시까지 무엇이 무엇을 막나)

```
BE 나머지 엔드포인트 배포 ─┐
BE Swagger(현재 2개) ──────┤
                          ▼
                    KB-66 API 공통 레이어 ──▶ KB-67~75 화면별 실연결 ──▶ 실기기 QA(안전검증) ──▶ 제출
디자인 롱폴(인트로/잠금/뱃지) ──▶ KB-76·78·84·25 UI ──────────────────────┘
```

1. **BE 엔드포인트 미배포가 최대 병목.** 현재 Swagger엔 스캔·상세 2개만. 인증·프로필·홈·검색·리뷰·랭킹 계약 미정의 → 실연결 다수 블록.
2. **KB-66(공통 레이어)** = 실연결 9개의 선행. 응답 래퍼 `{success,payload,message}` 확정으로 착수 가능(진행 중).
3. **디자인 롱폴** = 인트로 3슬라이드+일러스트, 비회원 잠금(상세/리뷰), 위험도 4-state 뱃지. 발주 완료(2026-07-06), 수령 후 UI 착수.
4. **안전 검증(SC-003 false-safe=0)** = BE 실위험도 붙은 뒤 실기기 QA. 출시 직전 관문.

- **BE 완성도**: 현재 실 엔드포인트 2개뿐. 나머지(인증·프로필·홈·검색·리뷰·랭킹) 배포 일정이 출시 최대 변수. → 종한과 일정 확정 필요(§7 플래그 후보).
