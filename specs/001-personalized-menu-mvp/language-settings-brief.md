> 트리거: 스캔 결과가 앱 언어를 무시하고 영어로 응답(A90·S22 재현) → 뿌리 = DB appLanguage

## 원칙 (한 줄)

**앱 언어의 유일한 정본 = 기기/OS 언어. 서버는 언어를 저장하지 않고, 매 요청의 `lang` 파라미터로만 안다.**

## 근거

- **정합성 구조 해소**: 언어를 저장하는 곳이 0곳 → 기기 간 충돌이 원리적으로 불가능.
- **글로벌 관행 일치**: 주요 글로벌 앱 전부 기기 언어 자동 + OS 앱별 설정 위임.
- **타깃 적합**: 한국 거주 외국인은 폰이 이미 자기 언어 — 기기 언어가 강한 신호.
  감지가 틀린 유저(예: 영어 폰 일본인)는 프로필>언어 행 → OS 설정으로 복구.
- **부수 이득**: OS 앱별 언어 변경 시 iOS·안드 모두 앱 재시작 → 언어 전환 캐시 무효화
  로직(P-015·P-018 계열) 대폭 단순화.
- **출시 전 유일 시점**: 구버전 사용자 = 팀 테플뿐이라 "즉시 완전 삭제"가 유일하게 싼 시점.

## 제외한 대안

- **인앱 피커 유지 + OS 선언 병행**: 정본 2곳(인앱 저장 vs OS)이라 개편 취지(정합성) 훼손 — 기각.
- **OS 정본 + 안드12- 전용 인앱 피커 잔존**: 분기 복잡도 대비 이득 적음 — 기각(안드12-는 폰 언어 추종).
- **온보딩 언어 화면을 안내문으로 존치**: 실기능 없는 스텝 — 기각.
- **appLanguage 수용-무시 후 추후 삭제**: 출시 전이라 하위호환 부담 없음 — 즉시 삭제 채택.
