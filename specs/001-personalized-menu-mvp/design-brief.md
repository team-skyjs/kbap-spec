# 디자인 브리프 — 1차 MVP (개인화 한식 메뉴 안전 안내)

> 이 문서는 디자이너(및 Claude design) 핸드오프용 요약본이다. 정본은 항상
> [`spec.md`](./spec.md)와 [헌법](../../.specify/memory/constitution.md)이며, 충돌 시 그쪽이
> 우선한다. UI 의도의 원천 디테일은 [`docs/최초-기획서.md`](../../docs/최초-기획서.md) 참고.

## 1. 제품 한 줄 요약

한국 여행/거주 외국인이 한식 **메뉴판을 촬영**하면, 자신의 알러지·종교·식이 프로필 기준으로
"먹어도 되는지"를 안전/주의/위험/판정불가로 안내받고, 음식 단위 정보·리뷰를 보고, '주의'
성분은 **사장님께 한국어로 확인**할 수 있는 모바일 앱. 정보는 '식당'이 아니라 '음식' 단위.

- 대상: 한국 방문/거주 외국인 (reader 언어 MVP=영어 우선)
- 제품명 **K-Bap** (표시명; 기술 슬러그 `kbap`), 코드네임 kfood
- 플랫폼: 모바일 앱 (iOS/Android)

## 2. 절대 지켜야 할 디자인 제약 (헌법)

1. **기본 이모지 사용 금지.** 위험도 신호 포함 모든 아이콘은 **디자인된 아이콘 세트**(아이콘
   폰트/SVG)로. (🔴🟡🟢 같은 OS 이모지 금지 — 플랫폼별 렌더링 불일치)
2. **다국어(i18n) 내성.** 모든 텍스트는 번역으로 길이가 늘어난다(영어·태국어 등). 레이아웃은
   **가변 길이 텍스트**를 견디고 잘리지 않아야 한다. 고정폭 가정 금지.
3. **위험도 4상태**를 색만이 아니라 **형태(아이콘 모양)로도** 구분 — 색맹 접근성.
4. **두 언어 축 분리:** 사용자가 읽는 UI(reader=영어 등) vs 사장님께 보여줄 문구(place=한국어).
   사장님 확인 화면의 한국어는 크고 명확하게.

## 3. 디자인 시스템 먼저 (화면보다 우선 정의)

화면을 그리기 전에 아래 공통 요소를 1세트로 확정해 전 화면에 일관 적용한다.

- **위험도 4상태 비주얼 언어** (가장 중요):
  - 안전(safe) / 주의(caution) / 위험(danger) / 판정불가(unable-to-assess)
  - 각 상태에 고유 **색 + 아이콘 형태 + 라벨**. 메뉴 목록의 작은 배지부터 상세의 큰 표시까지
    동일 언어로 스케일.
- **색 토큰** (브랜드 미정이므로 와이어프레임에선 그레이스케일 + 위험도 4색만)
- **타이포그래피** (다국어 폰트 고려: 라틴 + 태국 등 확장 대비)
- **공통 컴포넌트:** 별점(5점), 국적 칩(국기), 랭킹 배지(입문자~뼛속까지 한국인), 빈/로딩/에러
  상태, 바텀시트, 5탭 바(가운데 카메라 강조), 헤더(좌 로고 / 우 알림·검색)

## 4. 화면 목록 & 화면별 요구사항

> 각 화면은 Claude design에서 **개별 생성**하되, 위 디자인 시스템을 공통 컨텍스트로 먼저 제공.
> 괄호의 FR/US는 spec.md 추적용.

### A. 온보딩 (US1)
- 가입: 이메일 인증 / 애플 / 구글 (FR-001), 닉네임·국적·언어 필수 (FR-002)
- 식이 제약 입력 — 알러지/비건·채식/종교 **필수, 건너뛸 수 없음** (FR-003)
- 매운맛 0~10 (선택) — 단계마다 외국인 친화 비유(예: 7=불닭볶음면) (FR-004)
- 관심 한식 (선택) — 인기순 9개 → 선택 시 연관 9개 펼침(스포티파이식) (FR-005)
- **안전 고지 1회 동의** 단계 (FR-030)

### B. 앱 셸 (FR-026)
- 5탭: 홈 / 음식 / **카메라(가운데 크게·강조)** / 커뮤니티(자리만, 비활성) / 프로필
- 헤더: 좌 로고, 우 알림·검색

### C. 홈 — 개인화 허브 (US6)
- 최근 스캔/조회 음식 + "리뷰 남기기" 유도 (FR-034)
- 추천 음식 + 프로필 기준 위험도 표시 (FR-035)
- '메뉴판 스캔' CTA / 신규 사용자 빈 상태 (FR-036)

### D. 카메라(렌즈) (US2)
- 촬영 + 좌하단 갤러리 업로드 (FR-007)
- 상단 모드 전환: 메뉴판/가공식품/음식 — **메뉴판만 활성**, 나머지 비활성/추후 안내 (FR-010)
- 스캔 진행 표시(위아래 스캔 애니메이션) (FR-009)
- 결과: 촬영 이미지 **위에 오버레이**(파파고/구글렌즈식)로 reader 언어 번역 + 각 항목 우측
  **위험도 배지(4상태)** (FR-008·011·031)
- **원본 ↔ 번역 오버레이 토글** 버튼 (FR-037)
- 항목 탭 → 음식 디테일 **바텀시트**

### E. 음식 디테일 (US2/US3) — 위→아래 순서 (FR-016)
1. 메뉴명(사용자 언어)
2. 전체 평점 + 같은 국적 평점(5점 별점) → 탭하면 리뷰 디테일
3. 설명(한국인 실사용 느낌, 영어 기준 150자 이내)
4. 음식 사진
5. 90%+ 재료/성분 — 위험/주의/안전 순
- '주의' 성분 탭 → 설명 + **'사장님께 확인' 버튼** (FR-017)
- **미등록 음식:** 번역만 + '판정불가' + "사장님께 확인" 경고 (FR-033)

### F. 사장님 확인 화면 (US3)
- place 언어=**한국어**로 "{메뉴명}에 {성분}이 들어가나요?" 크게 (FR-018)
- 하단에 "알러지가 있어 위험합니다" 설명
- 메뉴명은 메뉴판 인식 메뉴명과 **정확히 일치** (FR-019)

### G. 음식 탭 (US4)
- 음식 탐색 → 디테일 이동 (FR-020)
- 리뷰 디테일: 전체/내 국적 평점, 리뷰별 국기·랭킹, 내 언어 번역, 내 국적만 필터 (FR-023)

### H. 리뷰 작성 (US4)
- 5점 별점 + 본문 (FR-022)

### I. 프로필 (US5)
- 정보 CRUD(식이 정보 포함) (FR-006), 계정 삭제 흐름 (FR-032)
- 내 리뷰 모아보기 (FR-024)
- 랭킹 등급(입문자~뼛속까지 한국인) (FR-025)

### J. 공통 상태
- 로딩 / 빈 상태 / 에러·재시도 / 오프라인 / '판정불가' (Edge Cases)

## 5. 권장 진행 방식 (Claude design)

- **지금은 Wireframe(흑백 러프)부터.** 이유: 브랜드(이름·색·로고) 미정, 신규 UX 다수 →
  구조·플로우를 싸게 확정한 뒤 High Fidelity로. (High-fi는 브랜드 방향 결정 후)
- 순서: ① 디자인 시스템 골격(특히 위험도 4상태) → ② 와이어프레임 전 화면 → spec과 대조
  검증 → ③ 브랜드 방향 확정 후 High Fidelity.
- 생성 단위: 앱 전체를 한 번에 X. **위 화면(A~J) 단위로** 생성하되 디자인 시스템을 공통
  컨텍스트로 전달해 일관성 유지.
