# 주문+고지 원카드 기획안 (2026-07-21 확정)

> 배경: "안전 음식 상세 최하단 빈칸" 디스커버리(7/21) — 아이디어 16종 발산 후
> **K(알레르기 고지)+B(주문 카드) 결합**으로 수렴, 예진 확정. BE 무부담(기존 데이터만) 제약.
> 위치: 음식 상세 페이지 최하단 (safe/caution 위험도일 때. danger는 주문 유도 부적절 — 미노출,
> unable은 기존 '사장님께 확인' 카드가 그 자리 담당).

## 개념

안전 판정의 자연스러운 다음 행동 = 주문. 상세 하단 CTA "이 메뉴 주문하기" → 사장님에게
보여주는 **큰 한국어 카드** 한 장에 주문과 알레르기 고지를 함께 싣는다. 안전 고지가 주문에
무임승차 — 사장님이 주문 받는 순간 손님의 기피를 알게 된다.

## 화면 스펙

### 1) 진입 CTA (상세 최하단)
- 버튼: `이 메뉴 주문하기` (reader 언어, i18n ×10)
- 노출 조건: overallRiskStatus ∈ {safe, caution} — caution은 버튼 위에 기존 주의 문구 유지
- danger/unable: 미노출 (unable = 기존 확인 카드)

### 2) 사장님 카드 (풀스크린 — 기존 '사장님께 확인' 카드와 동일 패밀리)
- **place 언어 = 한국어 고정** (헌법: reader/place 2축)
- 본문 (큰 글자, 위→아래):
  1. `{음식 한국어명} {n}개 주세요.` — koreanName 사용, n = 수량 스테퍼(1~5, 기본 1)
  2. (프로필 기피 ∩ 없어도 **전체 기피** 표시 — K의 핵심) `저는 {재료 ko 나열}을(를) 못 먹어요. 들어가면 알려주세요.` — 81종 코드 → ko 라벨, 최대 표시 후 "외 n개" 접기
- 기피 0개: 2 문단 자동 생략 → 순수 주문 카드
- ~~③ 종교/식이 한 줄~~ **삭제(7/21 예진 확정)**: 회피 모델은 **평면 81종 재료로 통일**(카테고리·religion/diet 코드 폐기 — 7월 초 확정 결정). 종교·식이는 별도 축이 아니라 해당 재료들(돼지고기·알코올 등)이 기피 목록에 들어있는 것으로 표현됨 → ②문단이 이미 커버. BE 질의 불요.
- 하단(작게, reader 언어): 이 카드가 무슨 내용인지 사용자용 요약 + 닫기
- 수량 문구: 국·찌개 "1인분" 이슈는 MVP "1개" 통일 (분류 데이터 생기면 개선 — 백로그)

### 3) 데이터 (전부 기존 — BE 호출 0)
- 상세 응답: koreanName, overallRiskStatus / 프로필: avoidanceSubstanceCodes(평면 81종 — 종교·식이 별도 코드 없음), 코드→ko 라벨은 FE 81종 매핑 재사용
- 조사(을/를) 처리: ko 문장 템플릿에서 받침 분기 (FE 유틸)

## 헌법 체크
- 이모지 금지(아이콘 SVG) · i18n ×10(버튼·사용자 요약) · 카드 본문은 place=ko 고정(번역 아님)
- false-safe 무관(안전 판정 로직 불변 — 표시·행동 레이어)

## 확장 백로그 (출시 후)
- 프로필 탭 "내 알레르기 카드" 전역 진입점 (K 단독)
- 스캔 결과 화면 진입점 (타이밍 최적)
- TTS 읽어주기 (expo-speech — 재빌드)
- "1인분/1개" 음식 분류별 문구

## 가정·완화 (디스커버리 기록)
- 사장님이 화면을 읽어줌: 기존 확인 카드와 동일 가정(신규 위험 아님), 글자 크게·짧게
- 재료 ko명=주방 용어: 81종이 개별 재료 단위라 통용 판단
- 기피 0개 사용자: 고지 생략으로 순수 주문 카드
