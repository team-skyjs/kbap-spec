# 공부 브리프 남기기

## 목적/분업
- **이 세션(spec)**: 대화 맥락을 아는 상태에서 **브리프**(키워드·원리·간단 코드·확장 포인트)만 작성.
- **리서치 세션**: 브리프를 보고 `_templates/📝 공부노트 템플릿.md` 형식의 **상세 공부 노트**를 작성.
- **예진**: 완성된 노트로 공부.

## 위치 규칙
- 볼트: `/Users/yejinkim/dev/local-handoff/study/`
- 브리프는 **`_briefs/YYYY-MM-DD-<주제슬러그>.md`** 에 남긴다 (리서치 세션의 인박스).
- 브리프 안에 "완성 노트가 들어갈 폴더"를 지정: 기존 폴더(`Expo & React Native` / `백엔드` / `MCP` / `Codex 작동원리` 등) 중 적합한 곳. 없으면 새 폴더 제안.

## 브리프 형식 (이 구조 고정)
```markdown
# [브리프] <주제 한 줄>

> 작성: YYYY-MM-DD (실제 date 출력) · 발신: spec 세션 · 대상: 리서치 세션
> 완성 노트 위치: study/<폴더>/<제안 파일명>.md · 템플릿: _templates/📝 공부노트 템플릿.md

## 무슨 일이 있었나 (맥락 — 짧게)
실제 겪은 현상/문제/결정. 프로젝트 맥락 1~3문장.

## 핵심 키워드 (리서치 시드)
- 검색 가능한 용어 위주로 5~10개 (영문 병기)

## 공부해야 할 원리
- 이 현상을 이해하는 데 필요한 개념을 질문 형태로 3~6개

## 간단 코드 / 재현
실제 코드·로그 발췌 (문제의 최소 재현이 보이게)

## 리서치 세션에게 — 확장 포인트
- 상세 노트에서 다뤄줬으면 하는 것 (비교/트레이드오프/베스트 프랙티스)

## 관련 파일 (프로젝트)
- 실제 코드/문서 경로
```

## 원칙
- **브리프는 짧게** — 상세 설명·긴 이론은 리서치 세션 몫. 맥락과 방향만 정확히.
- 날짜는 실제 `date` 출력 사용.
- 한 브리프 = 한 주제. 여러 주제면 파일 분리.
- 작성 후 예진에게 경로 + 리서치 세션에 넘길 한 줄 프롬프트를 알려준다.
