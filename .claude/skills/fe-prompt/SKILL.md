# FE 세션 프롬프트 설계

**왜 프롬프트에 다 실어야 하나**: 구현 세션(kbap-fe)은 이 레포의 스킬·문서를 자동으로 읽지
않는다. **프롬프트가 유일한 전달 경로**다 (jira 스킬 §5 경고).

## 프롬프트에 반드시 포함 (순서대로)
1. **작업 정의**: Jira 이슈 **키+작업명**(예: `KB-70 음식 상세 실연결`) + 본문의
   Background/DoD 요지 — 이슈를 조회해 요약해 넣는다. **비범위도 명시**(스코프 크립 방지).
2. **API 규격**: 계약 SSOT = **BE Swagger** `https://meogo.handev.site/swagger-ui` ·
   응답 래퍼 `{success,payload,message}` · `success=false`로 분기(HTTP 상태 아님, payload null
   방어) · **BE `message`를 사용자에게 노출 금지**(FE 자체 i18n으로 렌더).
3. **완료 게이트**: 작업이 끝나면 Jira를 **`검토 중`(전환 31)으로만** 전환하고 진행로그 댓글
   (날짜/진행한 것/블락된 것/진행할 것 — 요지 1~3줄)을 남긴다. **`완료` 전환 금지** — 리뷰는
   커맨드 센터(예진+Claude)가 한다.
4. **헌법 게이트**: 기본 이모지 0(SVG만) · i18n 하드코딩 0(9개 언어) · 위험도 4상태
   색/형태 고정 · **false-safe 0** — 불확실하면 unable/caution으로 강등, 근거 없는 safe 금지.
5. **진행 규칙**: PROGRESS.md(또는 kbap-fe의 진행 파일) 갱신 · **한 번에 1단위**만 ·
   막히거나 결정이 필요하면 임의 가정 말고 질문.

## 출력 형식
- 완성 프롬프트를 `----------` 구분선으로 감싼다(복붙용). 구분선 밖에 맥락 설명.
- 이슈 여러 개면 프롬프트를 분리(세션 컨텍스트 비대화 방지 — fe-handoff §12 교훈).

## 원칙
- 프롬프트가 인용하는 계약·정책은 항상 **정본**(Swagger·local-handoff 정책 문서·헌법 v2.0.0).
  fe-handoff §9 구템플릿(목업 단계용)을 쓰지 않는다.
