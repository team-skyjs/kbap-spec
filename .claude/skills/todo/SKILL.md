## 타임스탬프 규칙 (중요)
시각은 **절대 추측하지 말고** Bash로 실제 시각을 받아서 쓴다:
```
date '+%Y-%m-%d %H:%M %Z'
```
이 출력값을 그대로 타임스탬프로 사용.

상태 표기: ⬜ 대기 · 🔄 진행중 · ✅ 완료 · ⏸ 파킹

## 트리거 A — "뭐 해야하지 / 뭐하고있었지 / 다음 / 이어서 / 어디까지"
1. **LAUNCH.md 전체를 Read** (특히 In-Progress 큐 + 🔄 항목 + ⏸ 파킹).
2. **메모리도 스캔** (MEMORY.md → 관련 항목).
3. 다음 형식으로 간결히 보고:
   - **🔄 진행 중:** (있으면 무엇을, 언제 시작했는지)
   - **🔜 다음 큐:** 우선순위 순 몇 개
   - **⏸ 파킹:** 잠시 미뤄둔 것
   - **✅ 최근 완료:** 1~2개(맥락용)
4. "이거부터 할까?" 하고 이어갈 지점을 제안.

## 트리거 B — 할 일 추가 ("~ 할 일로 적어둬 / 추가해줘")
1. `date '+%Y-%m-%d %H:%M %Z'` 실행.
2. LAUNCH.md 적절한 섹션(보통 In-Progress 큐 또는 해당 카테고리)에 추가:
   `- [ ] <할 일> — 🆕 added <timestamp>`
3. 상단 `최종 갱신` 날짜 갱신.

## 트리거 C — 할 일 시작 ("~ 시작할게 / 이거 할게")
1. `date` 실행.
2. 해당 항목을 진행중으로: `- [ ] <할 일> — 🔄 started <timestamp>` (added 있으면 뒤에 유지).
3. 상단 `최종 갱신` 갱신.

## 트리거 D — 할 일 완료 ("~ 끝냈어 / 완료 / done")
1. `date` 실행.
2. 항목 체크 + 완료 시각: `- [x] <할 일> — ✅ done <timestamp>` (started/added 이력 있으면 뒤에 짧게 유지).
3. In-Progress 큐에 있던 항목이면 큐에서 빼고 `완료됨 ✅` 섹션(또는 해당 위치)에 반영.
4. 상단 `최종 갱신` 갱신.
