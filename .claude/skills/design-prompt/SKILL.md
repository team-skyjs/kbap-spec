# 디자인 프롬프트 설계

## 절차
1. **정본 수집**: 대상 화면의 FR을 `spec.md`에서, 화면 요구를 `design-brief.md`에서 확인.
   기존 디자인 맥락 = Claude Design 프로젝트(Direction G — fe-handoff §2·§4). 모호하면
   임의 확정하지 말고 사용자에게 먼저 질문.
2. **프롬프트에 반드시 포함**:
   - 제품 한 줄(K-Bap, 외국인 대상 한식 안전 안내) + 대상 화면의 목적/플로우 (**FR 번호 병기**)
   - **헌법 제약 4종**: ① 기본(유니코드) 이모지 금지 — 아이콘은 전부 SVG ② i18n 가변 길이
     내성(9개 언어, 고정폭 가정 금지) ③ 위험도 4상태(safe/caution/danger/unable)는 **색+형태**
     모두로 구분(색맹 접근성) ④ reader(UI 언어) / place(사장님 확인=한국어) 언어 축 분리
   - **Direction G 토큰**: primary `#E2580C` · surface `#FCF5EF` · 위험도 4색
     `#2f8f5b/#d28a12/#cf3a2c/#5b6470`(의미 고정, 변경 금지) — 상세는 fe-handoff §4
   - 생성 단위: **화면 1개씩**, 디자인 시스템(위험도 비주얼 언어 포함)을 공통 컨텍스트로 전달
3. **출력 형식**: 완성 프롬프트를 `----------` 구분선 한 줄로 위아래 감싼다(복붙용).
   구분선 밖에 맥락 설명·주의점을 짧게.

## 원칙
- 프롬프트가 인용하는 결정은 **최신 정본**(spec.md·헌법 v2.0.0·LAUNCH 회의 결정)이어야 한다.
  design-brief와 spec.md가 어긋나면 spec.md 우선 + design-brief 갱신 제안.
- 한 프롬프트 = 한 화면. 여러 화면이면 프롬프트 분리.
