## Q-01 ✅ 스캔 신계약 — 사진 업로드·가격·사진 전용 항목 (→ KB-72 완료 게이트)

[환경: dev+Metro — 콘솔 로그 확인 필요. 테플에서도 동작 자체는 확인 가능]

1. 로그인 상태에서 실제 메뉴판(가격 표기 있는 것) 스캔 ✅
2. ✅ 결과가 정상 표시되고, **Metro 터미널에** `[scan] upload-url issued` → `image upload complete` → `imagePath = scan/...` **실경로** 로그 ✅
3. ✅ 가격 표기된 메뉴에 **₩ 가격 표시**, 가격 없는 메뉴는 가격 영역 자체가 없음 ✅
4. ✅ 리스트 뷰에 **박스 없는 항목**(사진에서만 추출된 메뉴)이 있으면 위험도와 함께 노출 (없는 메뉴판이면 이 항목은 해당 없음 표시) ✅
5. (겸사) 한글+로마자 혼용 메뉴판 1회 — 김치찌개류 매칭 확인 ✅

### 피드백
 - 리스트화면에선 가격이 뜨는데 디테일 화면에선 가격 표시가 없어. detail api에도 가격 넣어주기로 종한이랑 계약했는데 혹시 api에 값이 오지 않는건지, 아니고 fe처리만 안한거라면 ui 추가해야해. 깨지지 않게 깔끔하게 가격 ui 보이도록 추가하게 브릿지에 작성해줘.
 - **[커맨드 센터 7/20]** 실측 결과 **BE 미배포**가 원인 (FoodDetailResponse에 price 없음 — 스펙·라이브 모두). FE 누락 아님. KB-179 생성 + P-012 등록(BE 배포 후 착수 게이트), 종한 리마인드 필요. 스캔 범위(KB-72)는 전 항목 통과라 완료 전환함.
 - **[커맨드 센터 7/20 확정]** 종한 확인 — detail API엔 가격 제공 불가(로직상)가 맞고, 설계적으로도 타당: **가격은 음식이 아니라 그 메뉴판(식당)의 속성** (같은 김치찌개도 식당마다 다름 — 스캔 계약에도 "응답 전용, 음식 마스터 미저장" 명시). 방향 확정: **스캔 결과→상세 이동 시에만 route param으로 가격 전달** (`/food/{id}?price=9000`), 홈·목록·북마크 진입은 가격 미표시가 정상. P-012 구현 완료(d134d1f, "스캔한 메뉴판 기준" 레이블) — **실기기 확인은 Q-12**.

4. 기내모드 켜고 사진 교체 시도 → ✅ 정직한 에러 문구 + 수정 화면 계속 사용 가능 (크래시 없음) ✅

- **[커맨드 센터 7/20]** 접수 — KB-149 범위 추가 + P-013 등록 (삭제 액션·플레이스홀더 복귀). 삭제의 PATCH 표현(null vs 빈 문자열)은 종한 확정 대기 — 확정되면 FE 착수. 구현 후 이 항목 5-1 재확인 예정.
- **[커맨드 센터 7/20 갱신 — 현재 상태]** 그 뒤 진행: ① P-013 완료(삭제 액션, 잠정 '') ② 예진×종한 확정 — 삭제=null 전송, 미설정/삭제 시 **서버가 기본 프로필 사진 URL 응답** ③ P-014 완료 — 기본 png 산출·dropbox 전달(예진이 종한 공유 완료), 삭제값 null 교체. **지금 대기 = 종한의 서버 배포**(기본사진 서빙+null 삭제). 배포되면 P-013+014 포함 OTA → 이 항목 6번으로 최종 확인. 그 전까지 Q-02는 🔄 유지 (1~5는 통과 기록됨).
- **[커맨드 센터 7/20 최종 확정]** 종한 답변으로 컨벤션 재확정 — null 폐기, **삭제·온보딩 미설정 = `images/default/profile/profile-default-512.png` 전송**, 응답도 null 없이 항상 URL. P-016 등록(상수 교체·온보딩 명시 전송·isDefault 판별로 삭제 버튼 노출 제어). P-016 완료+OTA 후 6번 확인 — 이때 "플레이스홀더 복귀"의 기대 결과는 **서버 기본 사진(png) 렌더**로 읽을 것.

## Q-03 ⬜ 맵기 -1 정책 (→ KB-150 후속 확인)

[환경: 테플 or dev]

1. 프로필 수정에서 맵기 설정 → Save → 앱 재실행 → ✅ 값 유지
2. "설정 해제" → Save → 앱 재실행/재조회 → ✅ **미설정 유지** (예전처럼 값이 되살아나지 않음)
3. ✅ 미설정 상태의 어느 화면에도 "-1" 노출 없음
4. 온보딩 신규 가입에서 맵기 **스킵** → 가입 후 프로필 → ✅ 미설정 표시 (FE가 실측 못 한 가정 — 어긋나면 ⚠️ 필수)

### 관련 결정 기록 (커맨드 센터 7/20)

- **4번이 존재하는 이유**: BE 정책은 "미설정 유저 = DB에 -1 저장"인데, 온보딩 **스킵 시 FE는 필드를 생략**해서 보냄 (P-003 판단 — 계약상 optional이라 생략이 미설정으로 수렴한다고 가정). FE 세션이 계정 생성 불가로 실측 못 해 실기기 확인으로 넘긴 것. **스킵했는데 프로필에 미설정이 아닌 값(예: 0)이 보이면 가정이 틀린 것** → ⚠️ 필수, FE가 "스킵 시 -1 명시 전송"으로 전환 (전환 지점 주석 있음).
- **기획 결정 (7/20 예진)**: "설정 해제" 기능은 **유지 확정** — 이미 구현·검증 완료라 제거 실익이 없고, 서버·로직은 어차피 미설정 상태를 지원해야 함. 대신 **노출을 덜 튀게 조정** (예: 별도 링크 대신 스케일 맨 앞 "설정 안 함" 위치) — KB-125 "맵기 UI 다듬기" 항목에서 처리 예정.

## Q-04 🔄 언어 5종 렌더 + 한·영 메뉴판 (→ KB-137 완료 게이트)

[환경: 테플 or dev]

1. 앱 언어를 ko → zh-Hans → zh-Hant → ja → th 순서로 변경하며 각각 홈·음식 상세·스캔 화면 훑기
2. ✅ 글자 깨짐(□·?) 없음
3. ✅ 제목 굵고 본문 얇은 **굵기 위계**가 살아 있음 (전부 같은 굵기면 ⚠️) ✅
4. [dev] 스캔 2회 → ✅ Metro에 `[scan] cleaned photo file` 로그 2회

### 피드백
1. 맵기 설명은 다 영어로만 되어있어. 어떤언어를 써도 모든 화면에서 영어로만 나와.
2. 언어를 바꾸고 음식탭에 들어가면 이전 언어의 음식명들이 나와. 음식 탭 클릭할때마다 새로 api를 받아오도록 변경해야할거같아.
3. 홈의 '시작하기 좋은 인기 요리' 이부분도 음식 이름들이 직전 언어로 나와. 갱신을 바로바로 해야할거같아. 그리고 처음에 한글 -> 다른언어로 바꿨을땐 한글만 나왔어. 동일한 원인인거같아 새로 갱신을 안해서. 근데 이것저것 다른거 하고(스캔 등) 홈으로 돌아오니까 제대로 설정 언어로 메뉴명들이 보여지고 있어. 암튼 탭 누를때마다 api 새로 받아와야할거같아. 
4. 한영 다 나와있는 메뉴판을 스캔했는데, 스캔 list화면에서 언어가 한글/영어만 나오고있어. api response에서 설정언어로 안보내주는건지 먼저 확인하고, 보내주고있는데 이렇게 보이는거라면 수정이 피룡해. detail페이지에는 내가 설정한 언어로 메뉴명들이 잘 나오고있어.
- **[커맨드 센터 7/20]** 4건 분류 완료: ①맵기 영어 고정 + ②③언어 전환 잔상 = **FE** → KB-187 생성 + P-015 등록 (라벨 i18n ×10 + 전환 시 무효화·스켈레톤, 원인 실조사 포함). ④스캔 name = **BE 계약 문제** — 스펙에 "현재는 항상 한국어" 명시(지역화 미구현), 상세는 lang 파라미터가 있어 정상. 종한에게 스캔 API lang 추가 질의 전달함(예진). BE 배포 후 FE 후속 여부 판단. Q-04의 KB-137 게이트(폰트 렌더 ②③)는 통과 — 4번(dev 스캔 로그)만 남음.

## Q-10 ⬜ [신호 대기] 토큰 만료·갱신 (종한 "30초/3분 배포" 알림 후)

[환경: 테플 or dev]

1. 로그인 → 30초 이상 대기 → 홈 당겨서 새로고침 → ✅ 에러 없이 조용히 갱신되어 데이터 표시
2. 3분 이상 방치 → 아무 조작 → ✅ 게스트로 강등 (로그인 화면 강제 아님 — 게스트 정책)
