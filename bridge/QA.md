## Q-12 ⬜ 스캔→상세 가격 표시 (→ KB-179 완료 게이트)

[환경: dev+Metro 지금 가능 / 테플은 P-011~013 묶음 OTA 후]

1. 가격 있는 메뉴판 스캔 → 결과에서 가격 표시된 메뉴 탭 → 상세 진입
2. ✅ 상세에 `₩n,nnn · 스캔한 메뉴판 기준` 한 줄 표시 (레이아웃 안 깨짐)
3. 같은 음식을 **음식 탭/검색에서** 진입 → ✅ 가격 미표시 (정상 — 어느 식당 가격인지 모르므로)
4. 가격 없는 메뉴를 스캔 결과에서 탭 → ✅ 가격 줄 없음

## Q-11 ⬜ 기피 재료 선택 UI B안 (→ KB-178 완료 게이트)

[환경: dev+Metro 지금 가능 / 테플은 P-011~013 묶음 OTA 후]

1. 온보딩 기피 재료 화면에서 목록 중간까지 스크롤 → 재료 여러 개 선택/해제
2. ✅ **목록이 밀리거나 스크롤 위치가 점프하지 않음** (핵심)
3. ✅ 새로 선택한 칩이 요약 줄 **끝에 나타나며 자동으로 보임**, 칩 ×탭으로 제거 가능
4. ✅ "계속 · n개 추가됨" 버튼이 스크롤 위치와 무관하게 **항상 하단에 노출**
5. 프로필 → 기피 재료 수정 화면도 동일 확인
6. 체감 판단: **0개→1개 첫 선택 순간의 1회 높이 변화**가 거슬리는지 — 거슬리면 ⚠️로 적어주세요 (placeholder 고정 높이로 전환 검토)

## Q-01 ✅ 스캔 신계약 — 사진 업로드·가격·사진 전용 항목 (→ KB-72 완료 게이트)

[환경: dev+Metro — 콘솔 로그 확인 필요. 테플에서도 동작 자체는 확인 가능]

1. 로그인 상태에서 실제 메뉴판(가격 표기 있는 것) 스캔 ✅
2. ✅ 결과가 정상 표시되고, **Metro 터미널에** `[scan] upload-url issued` → `image upload complete` → `imagePath = scan/...` **실경로** 로그 ✅
3. ✅ 가격 표기된 메뉴에 **₩ 가격 표시**, 가격 없는 메뉴는 가격 영역 자체가 없음 ✅
4. ✅ 리스트 뷰에 **박스 없는 항목**(사진에서만 추출된 메뉴)이 있으면 위험도와 함께 노출 (없는 메뉴판이면 이 항목은 해당 없음 표시) ✅
5. (겸사) 한글+로마자 혼용 메뉴판 1회 — 김치찌개류 매칭 확인 ✅

### 피드백
 - 리스트화면에선 가격이 뜨는데 디테일 화면에선 가격 표시가 없어. detail api에도 가격 넣어주기로 종한이랑 계약했는데 혹시 api에 값이 오지 않는건지, 아니고 fe처리만 안한거라면 ui 추가해야해. 깨지지 않게 깔끔하게 가격 ui 보이도록 추가하게 브릿지에 작성해줘.
 - **[커맨드 센터 7/20]** 실측 결과 **BE 미배포**가 원인 (FoodDetailResponse에 price 없음 — 스펙·라이브 모두). FE 누락 아님. KB-179 생성 + P-012 등록(BE 배포 후 착수 게이트), 종한 리마인드 필요. 스캔 범위(KB-72)는 전 항목 통과라 완료 전환함.

## Q-02 🔄 프로필 사진 — 업로드·렌더·유지 (→ KB-149 완료 게이트)

[환경: 테플 (OTA 019f7d42 이상 적용 — 앱 완전 종료 후 2회 실행)]

1. 프로필 수정 → 사진 선택(1:1 크롭) → ✅ 업로드 후 프로필 탭·수정 화면에 사진 렌더 ✅
2. ✅ 앱 완전 종료 → 재실행 → 사진 유지 (서버 조합 URL로 렌더) 
3. Metro 사용 시: ⚠️ `조회 profileImageUrl이 절대 URL이 아님` 로그가 보이면 피드백에 기록 (BE 확인 필요 신호) ✅ 이런 로그 안보임
4. 기내모드 켜고 사진 교체 시도 → ✅ 정직한 에러 문구 + 수정 화면 계속 사용 가능 (크래시 없음) ✅
5. 온보딩 신규 가입에서 사진 선택 → ✅ 가입 완료 후 프로필에 표시 (Q-06 계정 재사용 가능) ✅
6. [P-014 + 서버 기본사진 배포 후] "사진 삭제" 탭 → ✅ 즉시 기본 플레이스홀더 복귀 → **앱 완전 재시작 → 여전히 미설정** (⚠️ 사진이 되살아나면 서버가 삭제를 반영 안 한 것 — 피드백 필수, null 방식 전환 신호) → 사진 없는 상태에선 삭제 액션 자체가 안 보임

### 피드백
- 프로필 사진을 없앨수도 있어야해. 화면에 '프로필 사진 삭제' 누르면 우리 원래 사용하던 기본 svg(이미지?) 뜨도록 바꿔줘
- **[커맨드 센터 7/20]** 접수 — KB-149 범위 추가 + P-013 등록 (삭제 액션·플레이스홀더 복귀). 삭제의 PATCH 표현(null vs 빈 문자열)은 종한 확정 대기 — 확정되면 FE 착수. 구현 후 이 항목 5-1 재확인 예정.
